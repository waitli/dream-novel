# 优化方案：详细大纲 + 严格遵循 + 防截断

## 📋 问题分析

### 问题 1: 大纲不够详细 ❌
**现象**: 生成的章节大纲太简略，每章只有 1-2 句话  
**原因**: prompt 要求"每章字数控制在 100 字以内"  
**影响**: 写作时没有足够指导，AI 自由发挥导致偏离预期

### 问题 2: 写作没有严格遵循大纲 ❌
**现象**: 生成的章节内容与大纲不符，随意添加/改变情节  
**原因**: prompt 没有强制约束机制，缺少惩罚性说明  
**影响**: 大纲形同虚设，故事走向失控

### 问题 3: 前文摘要和角色状态被截断 ❌
**现象**: 写到 20-30 章后，前文摘要和角色状态更新不完整，部分内容丢失  
**原因**: 
- 字数限制太严格（2000 字）
- 没有结构化存储，AI 随意删减
- 缺少优先级机制，关键信息被误删

**影响**: 后续章节缺少上下文，出现逻辑矛盾、角色遗忘

## ✅ 优化方案

### 方案 1: 详细大纲（每章 200-300 字）

**修改文件**: `src/prompts/chapter-optimized.js`

**新增大纲字段**:
```
第 n 章 - [章节标题，10-20 字]
【本章定位】[20 字]
【核心作用】[30 字]
【情感基调】[15 字]
【出场角色】[50 字，包含状态变化]
【场景设计】[50 字，1-3 个场景]
【情节要点】
  1. 开场：[20 字]
  2. 发展：[40 字]
  3. 转折：[30 字]
  4. 收尾：[20 字]
【伏笔操作】
  - 埋设：[新伏笔及预计回收章节]
  - 强化：[已有伏笔加深]
  - 回收：[本章回收的伏笔]
【悬念密度】[★☆☆☆☆～★★★★★]
【情节张力】[★☆☆☆☆～★★★★★]
【关键台词】[1-2 句]
【本章简述】[200-300 字详细概括]
```

**对比**:
| 项目 | 原版 | 优化版 |
|------|------|--------|
| 每章字数 | 100 字 | 300-500 字 |
| 字段数量 | 5 个 | 12 个 |
| 情节要点 | 无 | 4 步详细分解 |
| 伏笔追踪 | 简单 | 埋设→强化→回收全链路 |
| 场景设计 | 无 | 包含感官细节 |

### 方案 2: 严格遵循大纲的写作 prompt

**修改文件**: `src/prompts/chapter-optimized.js`

**新增约束机制**:

```javascript
// firstDraft 和 nextDraft 中都添加了以下约束

## 写作要求（必须遵守）

### 1. 严格遵循大纲
- **必须按照【本章简述】中的情节要点展开**
- 不得随意添加大纲中没有的重大情节
- 不得改变大纲中设定的角色关系和事件走向
- 如有必要调整，必须在前文有合理铺垫

### 2. 上下文连贯性（后续章节）
- **与前文摘要保持一致**，不得出现矛盾
- **与前章结尾段自然衔接**，过渡流畅
- **角色状态一致**，性格、能力、关系符合已有设定
- 为下一章做好铺垫（参考【下一章预告】）

### 3. 场景设计（至少 2 个场景）
- 对话场景：体现人物性格和关系
- 动作/互动场景：环境交互细节，感官描写
- 心理/情感场景：内心活动，情感变化
- 环境场景：氛围营造
```

**惩罚性说明**:
```
**警告**: 如果偏离大纲，会导致故事逻辑混乱、角色 OOC、伏笔断裂。
**必须**严格遵循大纲中的情节要点、角色设定和伏笔安排。
```

### 方案 3: 结构化摘要和角色状态（防截断）

**修改文件**: `src/prompts/utility-optimized.js`

#### 3.1 前文摘要（结构化 + 优先级）

**新结构**:
```markdown
### 【核心主线】(300 字)
- 故事核心冲突和主角目标
- 当前进展阶段
- 主要对手/障碍

### 【重要事件】(500 字)
按时间顺序列出关键事件：
1. 第 1-10 章：[重大事件 1-3]
2. 第 11-20 章：[重大事件 4-6]
...

### 【角色关系】(400 字)
- 主角与关键角色的关系变化
- 重要阵营/势力的立场
- 未解决的关系冲突

### 【未解之谜】(300 字)
- 已埋设但未回收的伏笔
- 悬念和待揭示的真相
- 读者期待解答的问题

### 【世界观揭示】(300 字)
- 已揭示的世界规则
- 特殊能力/物品设定
- 地理/历史背景
```

**字数限制**: 3000 字（原 2000 字）

**优先级机制**:
```
如果超过字数限制，优先保留：
核心主线 > 重要事件 > 角色关系 > 未解之谜 > 世界观揭示
```

---

#### 3.2 角色状态（表格化 + 自动清理）

**新结构**:
```markdown
### 【主角】(100 字/人)
[角色名] | 状态：[活跃/离场/死亡] | 章节：[初登场→当前]
- 物品：[关键物品 1-3 个，每件 10 字]
- 能力：[核心能力 1-3 个，每个 15 字]
- 状态：[身体状态 20 字 | 心理状态 30 字]
- 关系：[关键关系 2-3 个，每个 15 字]
- 目标：[当前目标 30 字]
- 变化：[本章变化 30 字]

### 【重要配角】(80 字/人)
[角色名] | 状态：[活跃/离场/死亡] | 章节：[初登场→最后出现]
- 身份：[角色身份 20 字]
- 关系：[与主角关系 20 字]
- 状态：[当前状态 20 字]
- 变化：[本章变化 20 字]

### 【角色关系网】(200 字)
用箭头表示关系变化：
- [角色 A] → [信任] → [角色 B]（第 X 章建立）
- [角色 C] → [敌对] → [角色 D]（第 Y 章激化）

### 【淡出角色】(50 字)
简要列出已淡出视线的角色：
- [角色名]：第 X 章后未出现，原因 [...]
```

**自动清理机制**:
```
1. 对于 30 章以上未出现的角色，移入【淡出角色】
2. 新出场角色简要描述即可（50 字以内）
3. 关系变化必须标注章节
```

**字数限制**: 2500 字

#### 3.3 伏笔追踪（新增功能）

**专门文件**: `src/prompts/utility-optimized.js` - `trackForeshadowing`

**结构**:
```markdown
### 【新增伏笔】
- [伏笔名称]：第 X 章埋设，预计第 Y 章回收，描述 [...]

### 【强化伏笔】
- [伏笔名称]：第 X 章再次提及，线索加深 [...]

### 【回收伏笔】
- [伏笔名称]：第 X 章埋设 → 第 Y 章回收，效果 [...]

### 【待回收伏笔】(按优先级排序)
1. [高优先级] [伏笔名称]：第 X 章埋设，已超过 Y 章未回收
2. [中优先级] [伏笔名称]：第 X 章埋设，预计第 Z 章回收
3. [低优先级] [伏笔名称]：第 X 章埋设，刚埋设不久
```

**优先级规则**:
- 超过 30 章未回收：[高优先级]
- 超过 10 章未回收：[中优先级]
- 刚埋设不久：[低优先级]

在生成章节后，检查是否遵循大纲：

```javascript
async function checkChapterAlignment(chapter, blueprint, apiConfig) {

【大纲要求】
${blueprint}

【章节内容】
${chapter}

1. 情节要点：是否包含大纲中的开场/发展/转折/收尾

4. 伏笔操作：是否按大纲埋设/回收伏笔
5. 关键台词：是否包含或类似大纲中的关键台词

如有偏离，指出具体位置和偏离程度。
```

## 📝 总结

### 核心改进

1. **大纲详细度提升 4 倍**（100 字 → 400 字/章）
2. **写作约束力大幅增强**（多次强调 + 惩罚说明）
3. **摘要容量提升 50%**（2000 字 → 3000 字）
4. **结构化存储防截断**（优先级 + 自动清理）
5. **新增伏笔追踪功能**（埋设→强化→回收全链路）

### 预期效果

- ✅ 大纲能够真正指导写作
- ✅ 章节内容严格遵循大纲
- ✅ 长篇小说（50-100 章）不会出现上下文矛盾
- ✅ 伏笔不会遗忘或断裂
- ✅ 角色行为始终保持一致
